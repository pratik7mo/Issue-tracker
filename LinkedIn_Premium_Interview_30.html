30 Scenarios: Jenkins, ECR, EC2, and Production Troubleshooting

Jenkins & Pipeline Architecture
01 Jenkins build works but stage boxes are missing in Blue Ocean. How do you solve it?
Fix: This is almost always a structural error in the Declarative DSL. I refactored the pipeline from a messy script into a strict Declarative Structure with clear pipeline, stages, and steps blocks. This ensures the Jenkins parser can map the stages correctly to the UI.
02 Why use a local Jenkinsfile over a Shared Library?
Strategy: In this project, a local Jenkinsfile allowed the CI/CD logic to evolve directly with the code. It provided the team with immediate visibility and the ability to customize AWS deployment scripts without waiting for a central DevOps team to update a Shared Library.
03 How did you implement real-time CI triggers?
Insight: I configured GitHub Webhooks. Unlike polling (which is resource-heavy and slow), webhooks push an event to Jenkins the millisecond a commit hits the repo, triggering the pipeline instantly and providing developers with rapid feedback.
Docker & Containerization
07 How did you optimize the React Docker image size?
I used a multi-stage build. I compiled the React code in a heavy Node environment first, then copied only the production dist folder into a lightweight Nginx-Alpine image. This reduced the image from 1.2GB to just 22MB, making ECR pushes significantly faster.
08 How do containers communicate in your production stack?
I leveraged Docker Compose Networking. By placing the API, Database, and Redis containers on the same internal bridge network, they can communicate via their service names (e.g., db:5432), keeping the database off the public internet.
09 Tagging strategy: BUILD_NUMBER vs Latest?
In production, I always use Semantic Versioning with Build Numbers. Tagging images with ${env.BUILD_NUMBER} ensures that every deployment is traceable. If Build #45 breaks production, we can instantly rollback by pulling the image from Build #44.
Security & Credentials
13 How do you manage secrets for the production .env file?
I automate the creation of the .env file within the Jenkins workspace. Using Jenkins Credentials and the writeFile utility, I inject secrets into the file at runtime and then SCP it to the EC2 host. This keeps secrets out of Git entirely.
18 How do you deploy securely to EC2 without exposing keys?
I used the SSH Agent Plugin. By wrapping the scp and ssh commands in an sshagent(['id']) block, the private key is available only during the execution of that specific block in Jenkins memory and is never saved to the agent's disk.
AWS & Deployment Automation
19 How do you sync configurations to EC2?
In the 'Deploy' stage, I use SCP to transfer the docker-compose.prod.yml and .env files. This ensures the infrastructure configuration stays in sync with the application code changes in every build.
24 Why use Nginx as a Reverse Proxy?
Problem Solver: It resolves CORS issues. By proxying all /api requests to the backend container internally, the browser sees both frontend and backend as coming from the same host, eliminating complex cross-origin configuration.
Real-World Troubleshooting
25 How to fix a "JVM Out of Memory" during Maven builds?
I tuned the MAVEN_OPTS env variable in the Jenkinsfile. By setting -Xms256m -Xmx512m, I restricted the Maven process to a safe memory range, preventing it from consuming all agent resources and crashing the build.
28 What if the Database is not ready when the API starts?
I implemented Docker Healthchecks in the database container. Additionally, I configured the Spring Boot backend with a retry logic (or used a wait-for-it.sh script) to ensure it waits for the DB connection to be established before failing.
